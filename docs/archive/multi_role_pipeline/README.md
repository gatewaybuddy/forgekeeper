# Multi-Role Pipeline Deliverables

This directory captures the written artifacts called out in
`docs/plans/multi_role_pipeline.yaml`. Each file maps to a plan
"deliverable" so engineering, product, policy, and safety partners have a
common reference while implementation work proceeds.

| Plan Task | Deliverable | File |
|-----------|-------------|------|
| T1 | Role contract schema | `role_contract_schema.yaml` |
| T1 | Example contract instances | `example_role_contracts.yaml` |
| T2 | Service blueprint | `service_blueprint.md` |
| T2 | Authenticated action endpoints | `authenticated_action_endpoints.md` |
| T3 | Dataset refresh DAG | `dataset_refresh_dag.md` |
| T3 | Coverage gap report | `coverage_gap_report_template.md` |
| T4 | Feedback summary dashboard | `feedback_dashboard_spec.md` |
| T4 | Workflow SLA report | `workflow_sla_report_template.md` |
| T5 | Policy engine configuration | `policy_engine_configuration.md` |
| T5 | Automated rollback triggers | `automated_rollback_triggers.md` |
| T6 | Incident report templates | `incident_report_template.md` |
| T6 | Telemetry ingestion adapters (spec) | `telemetry_ingestion_adapters.md` |
| T7 | Provider contract documentation | `provider_contract_documentation.md` |
| T8 | Snapshot retention SOP | `snapshot_retention_sop.md` |
| T9 | Fence compliance checklist | `fence_compliance_checklist.md` |
| T10 | Sandbox usage playbook | `sandbox_usage_playbook.md` |
| T11 | Policy UX content guidelines | `policy_ux_content_guidelines.md` |
| T12 | Minimal repro SOP | `minimal_repro_sop.md` |
| T13 | Replay reproducibility guide | `replay_reproducibility_guide.md` |
| T14 | HITL escalation matrix | `hitl_escalation_matrix.md` |

Each document includes owners, review cadence, and references back to the
plan so future updates can stay synchronized.
